## 1. Scope

SP4b wires the Chainlit chat UI to all five crews so consultants can run any crew and respond to HITL gates directly in chat. All existing agents use the same HITL pattern ‚Äî produce output ‚Üí ask for approval/revision ‚Üí revise up to 3 times ‚Äî which maps cleanly onto `cl.AskUserMessage`.

**In scope:**
- `ChainlitHumanInputTool` ‚Äî async HITL tool using `cl.AskUserMessage`
- Registry injection ‚Äî `hitl_tool` param in `get_tools_for_agent()`
- Crew factory propagation ‚Äî `hitl_tool=None` in all 5 `create_*_crew()` functions
- Full `chainlit_app/app.py` rewrite ‚Äî project selection, crew menu, kickoff, completion

**Out of scope:**
- PAM orchestration ‚Äî SP4c
- n8n / Slack integration ‚Äî SP4c
- Per-agent bespoke Chainlit UI (forms, tables) ‚Äî deferred
- Agent progress streaming (step callbacks) ‚Äî deferred

## 2. Architecture

Chainlit runs the crew directly. The consultant selects a project and crew in chat, Chainlit calls `crew.kickoff_async()`, and HITL gates surface transparently as `cl.AskUserMessage` input widgets. No polling, no REST round-trips for the HITL path.

The REST API path (`POST /projects/{slug}/run`) is unchanged. `run_service.py` never passes `hitl_tool`, so the default `HumanInputTool` (SQLite polling) remains active for n8n/programmatic triggers.

```
Chainlit on_message
  ‚îî‚îÄ _run_crew(slug, crew_name)
       ‚îú‚îÄ create crew_run DB record ‚Üí run_id
       ‚îú‚îÄ ChainlitHumanInputTool(slug, run_id)
       ‚îú‚îÄ create_*_crew(..., hitl_tool=chainlit_tool)
       ‚îî‚îÄ await crew.kickoff_async()
            ‚îî‚îÄ ... agent steps ...
                 ‚îî‚îÄ ChainlitHumanInputTool._arun(prompt)
                      ‚îú‚îÄ insert_hitl_review(slug, run_id, prompt)  ‚Üê audit
                      ‚îú‚îÄ await cl.AskUserMessage(prompt).send()    ‚Üê Chainlit pause
                      ‚îú‚îÄ complete_hitl_review(slug, review_id, response)
                      ‚îî‚îÄ return response ‚Üí crew continues
```

`cl.AskUserMessage` does not go through `on_message` ‚Äî Chainlit handles it at the protocol level. No crew-running state detection is needed in the message handler.

## 4. ChainlitHumanInputTool

**File:** `agents/tools/chainlit_human_input.py`

Subclasses `HumanInputTool`. Overrides `_arun()` only. `_run()` is inherited (SQLite polling) for any synchronous callsite.

```python
import chainlit as cl
from agents.tools.human_input import HumanInputTool
from agents.tools._db import insert_hitl_review, complete_hitl_review

_DEFAULT_HITL_TIMEOUT = 86400  # 24 hours

class ChainlitHumanInputTool(HumanInputTool):
    name: str = "HumanInputTool"  # task descriptions match on name ‚Äî must not change

    async def _arun(self, prompt: str) -> str:
        try:
            review_id = insert_hitl_review(
                slug=self.slug, run_id=self.run_id, prompt=prompt
            )
        except Exception as e:
            return f"Error: could not create review record ‚Äî {e}"

        res = await cl.AskUserMessage(
            content=prompt, timeout=_DEFAULT_HITL_TIMEOUT
        ).send()
        response = res["output"] if res else "timeout: no response received"

        complete_hitl_review(
            slug=self.slug, review_id=review_id, decision=response
        )
        return response
```

**Key decisions:**
- `name = "HumanInputTool"` ‚Äî all task descriptions reference this name; must not change.
- `HITL_AUTO_RESPOND` env var is not checked in `_arun()` ‚Äî this tool is only used in live Chainlit sessions. Integration tests continue using `HumanInputTool` directly.
- SQLite audit record still written ‚Äî keeps the review trail consistent with the REST path.
- `complete_hitl_review` called on timeout too ‚Äî closes the record regardless.
- `insert_hitl_review` / `complete_hitl_review` are synchronous (`sqlite3`) ‚Äî fast point writes, no async needed.

## 7. Chainlit app.py Rewrite

**File:** `chainlit_app/app.py`

Session state in `cl.user_session`: `"slug"` (str | None), `"crew"` (str | None).

### on_chat_start

Reset session state. Send welcome message listing supported crews.

### on_message

Three phases based on session state:

**Phase 1 ‚Äî project selection** (`slug is None`):
- Validate slug via `GET /projects/{slug}/status` against FastAPI
- On 200: set `slug`, send crew menu
- On 404: send "Project not found"

**Phase 2 ‚Äî crew selection** (`crew is None`):
- Validate crew name against: `discovery | value_design | architecture | delivery | business_plan`
- Set `crew`, send "Starting‚Ä¶"
- `await _run_crew(slug, crew_name)`
- Reset `crew` to None (ready for another run on the same project)

**Phase 3 ‚Äî unexpected message while running** (else):
- This branch should not be reached ‚Äî `cl.AskUserMessage` handles HITL transparently
- Send: "Please respond to the agent prompt above."

### _run_crew(slug, crew_name)

```python
async def _run_crew(slug: str, crew_name: str) -> None:
    run_id: int | None = None
    try:
        run_id = await _create_run_record(slug, crew_name)

        from agents.tools.chainlit_human_input import ChainlitHumanInputTool
        hitl_tool = ChainlitHumanInputTool(slug=slug, run_id=run_id)

        config = load_project_config(Path(settings.projects_dir) / slug)
        llm_mode = config.get("llm_mode", "standard")
        sector   = config.get("sector", "")

        crew = _build_crew(crew_name, slug, run_id, llm_mode, sector, config, hitl_tool)

        await cl.Message(content=f"‚öô Running **{crew_name}** crew‚Ä¶").send()
        await crew.kickoff_async()

        await _mark_run_completed(slug, run_id, status="completed")
        await _send_completion_message(slug)

    except Exception as e:
        if run_id is not None:
            await _mark_run_completed(slug, run_id, status="failed")
        await cl.Message(content=f"‚úó Crew failed: {e}").send()
```

### _build_crew

Dispatch by crew name to the appropriate factory. Passes extra config params where needed (delivery crew requires `value_stream_labels`, `stakeholder_groups`, `roadmap_time_axis`).

### _send_completion_message

Reads `projects/{slug}/outputs/`, lists every file with name and human-readable size:

```
‚úì Crew complete. Outputs:
‚Ä¢ business_plan.docx  (142 KB)
‚Ä¢ executive_presentation.pptx  (88 KB)
‚Ä¢ cost_benefit_model.xlsx  (24 KB)
```

### _create_run_record / _mark_run_completed

Thin async wrappers around `get_connection` / `update_crew_run_status` from `api.database`. Use `aiosqlite` ‚Äî consistent with the API layer.
