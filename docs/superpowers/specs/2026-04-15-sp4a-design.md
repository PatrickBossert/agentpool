## 3. Agent — Business Plan Generator

### Identity
- **Role:** `"Business Plan Generator"`
- **Model:** Claude Opus 4.6 — `get_pam_llm(llm_mode)`
- **Tools:** `SQLiteStateTool`, `HumanInputTool`, `WordOutputTool`, `PowerPointOutputTool`, `FinancialModelTool`
- **allow_delegation:** False

### Inputs
All read via `SQLiteStateTool`:

| Key | Source crew |
|---|---|
| `requirements` | Discovery |
| `value_levers` | Discovery |
| `propositions` | Value Design |
| `initiative_register` | Architecture |
| `roadmap_data` | Delivery |

### Process (task description steps)

1. Read all five keys via SQLiteStateTool: `requirements`, `value_levers`, `propositions`, `initiative_register`, `roadmap_data`.
2. Pre-populate financial estimates from existing data:
   - **Initiative costs** — complexity_score brackets: 1=£50k, 2=£100k, 3=£200k, 4=£400k, 5=£800k
   - **Annual benefits** — value_estimate: High=£500k/yr, Medium=£200k/yr, Low=£100k/yr
   - **Discount rate** — default 8%
   - **Period duration** — inferred from `roadmap_data.time_axis`: quarters=3 months, years=12 months, horizons=18 months
3. Use `HumanInputTool` to gather and confirm business context and financial assumptions: *"To complete the business plan, please provide: (1) Organisation name, (2) Financial year (e.g. FY2026), (3) Primary business sponsor name and title, (4) Any additional context for the executive summary. I've also pre-populated financial estimates — please confirm or adjust: [list each initiative with estimated cost, each proposition with estimated annual benefit, and the default 8% discount rate]. Reply with your details and 'confirmed' or provide adjustments."*
4. Generate all six business plan sections using LLM reasoning:
   - Executive Summary (incorporating organisation name, sponsor, financial year, business context provided)
   - Case for Change (from requirements + value_levers)
   - Value Propositions (from propositions register)
   - Initiative Roadmap (from roadmap_data — periods, initiatives by period, capability builds)
   - Investment & Benefits (from financial model inputs — costs by period, benefits from realisation, NPV/IRR summary)
   - Governance & Next Steps
5. Call `WordOutputTool` with `sections` list, `metadata`, `filename='business_plan.docx'`, `agent_name='business_plan_generator'` → `outputs/business_plan.docx`.
6. Assemble 8–10 slides and call `PowerPointOutputTool` with `slides` list, `metadata`, `filename='executive_presentation.pptx'`, `agent_name='business_plan_generator'` → `outputs/executive_presentation.pptx`.
7. Assemble confirmed financial inputs and call `FinancialModelTool` with `periods`, `initiatives` (with cost_gbp), `propositions` (with annual_benefit_gbp), `discount_rate`, `period_duration_months`, `filename='cost_benefit_model.xlsx'`, `agent_name='business_plan_generator'` → `outputs/cost_benefit_model.xlsx`.
8. Use `HumanInputTool` with prompt: *"Please review the outputs: outputs/business_plan.docx, outputs/executive_presentation.pptx, and outputs/cost_benefit_model.xlsx. Reply 'approved' to conclude Business Plan generation, or provide revision notes."*
9. If revision notes received, revise and repeat steps 5–8. Maximum 3 revision cycles.

### Outputs
- `outputs/business_plan.docx` — via WordOutputTool
- `outputs/executive_presentation.pptx` — via PowerPointOutputTool
- `outputs/cost_benefit_model.xlsx` — via FinancialModelTool

## 4. Tool — WordOutputTool

**File:** `agents/tools/word_output.py`
**Pattern:** `BaseTool` subclass — identical structure to `ExcelOutputTool`

### Input Schema
- `sections` — list of `{"title": str, "content": str}` dicts — one per business plan section
- `metadata` — dict with keys: `org_name`, `financial_year`, `sponsor`, `prepared_by` (optional), `date` (auto-filled if absent)
- `filename` — output filename (`.docx` extension added automatically if missing)
- `agent_name` — for output tracking

### Behaviour
- Generates a styled `.docx` using `python-docx`:
  - Title page: org_name, "Digital Modernisation Business Plan", financial_year, date, sponsor
  - Static table of contents listing section titles
  - One `Heading 1` + body paragraph per section from the `sections` list
  - Styling: Arial font, navy headings
- Writes to `projects/{slug}/outputs/{filename}`
- Calls `insert_agent_output_sync` with `output_type="docx"`
- Returns absolute file path on success; `"Error: render failed — {reason}"` on failure

### Extensibility
The `sections` list is open-ended — the agent passes any number of sections with any titles. No structural change to the tool is needed if the business plan gains new sections in future.

## 5. Tool — PowerPointOutputTool

**File:** `agents/tools/powerpoint_output.py`
**Pattern:** `BaseTool` subclass — same structure as `WordOutputTool`

### Input Schema
- `slides` — list of `{"title": str, "content": str | list[str], "notes": str}` dicts — one per slide
- `metadata` — dict with keys: `org_name`, `financial_year`, `sponsor`, `date`
- `filename` — output filename (`.pptx` extension added automatically if missing)
- `agent_name` — for output tracking

### Behaviour
- Generates a `.pptx` using `python-pptx`:
  - Slide 1 (title layout): org_name, presentation title, financial_year, date
  - Slides 2–N (content layout): title from `slides[i].title`, bullet points from `slides[i].content` (if list) or paragraph text (if string), speaker notes from `slides[i].notes`
  - Consistent theme: dark navy title bar, white body, Arial font
- Writes to `projects/{slug}/outputs/{filename}`
- Calls `insert_agent_output_sync` with `output_type="pptx"`
- Returns absolute file path on success; `"Error: render failed — {reason}"` on failure

### SP4a slide set (assembled by agent — 8–10 slides)

| # | Title | Content source |
|---|---|---|
| 1 | Title | metadata |
| 2 | Case for Change | requirements + value_levers summary |
| 3 | Our Value Propositions | propositions (high value first) |
| 4 | Delivery Roadmap | roadmap_data periods + initiative summary |
| 5 | Investment & Benefits | financial model summary (NPV, IRR, max borrowing) |
| 6 | Governance & Next Steps | generated from business context |

### Extensibility
The `slides` input is a generic list — the agent decides what to pass. A future richer deck passes more slides with more detailed content (initiative tables, roadmap image embeds) without any tool interface change.

## 6. Tool — FinancialModelTool

**File:** `agents/tools/financial_model.py`
**Pattern:** `BaseTool` subclass — same structure as `WordOutputTool`

### Input Schema
- `periods` — list of period name strings (e.g. `["Q1 2026", "Q2 2026", "Q3 2026"]`) from `roadmap_data.periods`
- `initiatives` — list of `{"id": str, "title": str, "period": str, "cost_gbp": float}`
- `propositions` — list of `{"id": str, "title": str, "realisation_period": str, "annual_benefit_gbp": float}`
- `discount_rate` — float (e.g. `0.08` for 8%)
- `period_duration_months` — int (e.g. `3` for quarterly)
- `filename` — output filename (`.xlsx` extension added automatically if missing)
- `agent_name` — for output tracking

### Behaviour
- Writes a 3-sheet `.xlsx` using `openpyxl` (already in requirements):

**Sheet 1 — Cashflow Model:**
- Columns: row label + one column per period
- Rows: one per initiative (cost in its period, zero elsewhere); one per proposition (recurring annual benefit pro-rated by period_duration_months, starting from realisation_period onwards); Net Cashflow row (benefits − costs per period); Cumulative Cashflow row (running total)

**Sheet 2 — Financial Summary:**
- NPV — calculated in Python: `sum(cf / (1 + rate_per_period) ** t for t, cf in enumerate(cashflows))`
- IRR — calculated in Python via binary search iteration (no external dependencies)
- Payback Period — first period where cumulative cashflow turns positive
- Maximum Borrowing Requirement — `min(cumulative_cashflows)` — the peak funding need before returns begin
- Total Investment — sum of all initiative costs
- Total Benefits (over model horizon) — sum of all benefit cash flows

**Sheet 3 — Assumptions:**
- Discount rate, period duration (months), cost and benefit inputs used

- Writes to `projects/{slug}/outputs/{filename}`
- Calls `insert_agent_output_sync` with `output_type="xlsx"`
- Returns absolute file path on success; `"Error: render failed — {reason}"` on failure

### IRR Calculation (no external dependencies)
```python
def _calculate_irr(cashflows: list[float], max_iterations: int = 1000) -> float | None:
    """Binary search IRR. Returns None if no solution found."""
    low, high = -0.999, 10.0
    for _ in range(max_iterations):
        mid = (low + high) / 2
        npv = sum(cf / (1 + mid) ** t for t, cf in enumerate(cashflows))
        if abs(npv) < 0.01:
            return mid
        if npv > 0:
            low = mid
        else:
            high = mid
    return None
```
