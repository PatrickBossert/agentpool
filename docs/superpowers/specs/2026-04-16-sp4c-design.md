## 1. Scope

SP4c wires the five specialist crews into a single end-to-end pipeline driven by PAM (Programme Architecture Manager), a CrewAI agent (Opus 4.6). A consultant triggers a full pipeline run via a Slack slash command or direct API call. PAM runs each crew in sequence, notifies Slack when each completes, and the within-crew HITL gates (SP4b) surface naturally in the Chainlit session as before.

**In scope:**
- `RunCrewTool` — tool that runs a named sub-crew and awaits completion
- `SlackNotifyTool` — tool that POSTs a notification event to the n8n webhook
- PAM agent + 5 sequential tasks (`agents/pam/pam_agent.py`)
- PAM crew factory (`agents/crews/pam_crew.py`)
- `POST /projects/{slug}/orchestrate` endpoint + orchestration service
- `orchestration_runs` DB table
- `run_service.py` refactor — extract shared `build_and_run_crew` helper
- Two n8n workflow JSON files (`/workflows/`)
- Slack app setup instructions

**Out of scope:**
- Chainlit trigger for PAM (PAM is REST/n8n only; Chainlit continues to run individual crews via SP4b)
- Inter-crew blocking review gates (within-crew HITL in SP4b covers artefact approval)
- PAM re-run / retry logic
- SP2 web platform

## 2. Architecture

```
POST /projects/{slug}/orchestrate
  └─ orchestration_service.start_orchestration(slug)
       ├─ insert_orchestration_run → orchestration_run_id
       └─ asyncio.create_task(run_pam_crew(slug, orchestration_run_id))
            └─ PAM crew (Opus 4.6, Process.sequential, 5 tasks)
                 ├─ Task 1: RunCrewTool("discovery")
                 │          SlackNotifyTool("✓ Discovery complete…")
                 ├─ Task 2: RunCrewTool("value_design")
                 │          SlackNotifyTool("✓ Value Design complete…")
                 ├─ Task 3: RunCrewTool("architecture")
                 │          SlackNotifyTool("✓ Architecture complete…")
                 ├─ Task 4: RunCrewTool("delivery")
                 │          SlackNotifyTool("✓ Delivery complete…")
                 └─ Task 5: RunCrewTool("business_plan")
                            SlackNotifyTool("✓ Pipeline complete. All outputs ready.")
```

The five specialist crews are unchanged. Each `RunCrewTool._arun()` call blocks until the sub-crew's `kickoff_async()` returns — including any within-crew HITL waits. PAM simply waits for the tool to return before moving to the next task.

The REST API's existing `POST /projects/{slug}/run` endpoint for individual crew dispatch is untouched. Chainlit continues to use the SP4b path.

## 4. RunCrewTool

**File:** `agents/tools/run_crew.py`

Pydantic fields: `slug: str`, `orchestration_run_id: int`.

```python
class RunCrewTool(BaseTool):
    name: str = "RunCrewTool"
    description: str = (
        "Run a named crew for the current project and wait for it to complete. "
        "crew_name must be one of: discovery, value_design, architecture, delivery, business_plan"
    )
    slug: str
    orchestration_run_id: int

    async def _arun(self, crew_name: str) -> str:
        from api.database import get_connection, fetch_project, insert_crew_run, update_crew_run_status
        from api.services.run_service import build_and_run_crew

        async with get_connection(self.slug) as conn:
            project = await fetch_project(conn, slug=self.slug)
            run_id = await insert_crew_run(
                conn, project_id=project["id"], crew_name=crew_name, status="running"
            )
        try:
            result = await build_and_run_crew(self.slug, crew_name, run_id)
            async with get_connection(self.slug) as conn:
                await update_crew_run_status(conn, run_id=run_id, status="completed")
            return str(result)
        except Exception as e:
            async with get_connection(self.slug) as conn:
                await update_crew_run_status(conn, run_id=run_id, status="failed")
            return f"Error running {crew_name}: {e}"
```

`_run()` is not used — PAM crew runs via `kickoff_async()` which calls `_arun()`.

**Key decisions:**
- No `hitl_tool` passed to crew factories — the REST path uses the default SQLite-polling `HumanInputTool`
- `build_and_run_crew` is extracted from `run_service.py` (see section 7)
- Returns an error string on failure rather than raising — PAM can see the error in context and the pipeline continues (or PAM surfaces it in the final Slack notification)

## 5. SlackNotifyTool

**File:** `agents/tools/slack_notify.py`

Pydantic field: `slug: str`.

```python
class SlackNotifyTool(BaseTool):
    name: str = "SlackNotifyTool"
    description: str = "Send a notification message to the project's Slack channel via n8n."
    slug: str

    def _run(self, message: str) -> str:
        from api.config import get_settings, load_project_config
        from pathlib import Path
        settings = get_settings()
        if not settings.n8n_webhook_url:
            return "notification skipped (no webhook configured)"
        try:
            config = load_project_config(Path(settings.projects_dir) / self.slug)
            slack_channel = config.get("slack_channel", "")
            import httpx
            httpx.post(
                settings.n8n_webhook_url,
                json={
                    "event_type": "crew_notification",
                    "project_slug": self.slug,
                    "slack_channel": slack_channel,
                    "message": message,
                },
                timeout=5.0,
            )
            return "notification sent"
        except Exception as e:
            return f"notification failed (non-fatal): {e}"
```

Synchronous — `httpx.post` for a fast fire-and-forget. Never raises — a missing or unreachable webhook cannot crash the PAM crew.

The existing `HumanInputTool` already sends `event_type: "hitl_review"` to the same URL. n8n routes on `event_type` via a Switch node.

## 6. PAM Agent and Tasks

**File:** `agents/pam/pam_agent.py`

One agent factory and five task factories. Uses constants already defined in `agents/pam.py` (`PAM_MODEL`, `PAM_ROLE`, `PAM_GOAL`).

```python
def create_pam_agent(slug: str, llm: LLM, tools: list) -> Agent:
    return Agent(role=PAM_ROLE, goal=PAM_GOAL, llm=llm, tools=tools, ...)

def create_run_discovery_task(agent: Agent, slug: str) -> Task:
    return Task(
        description=(
            f"Use RunCrewTool with crew_name='discovery' to run the Discovery crew "
            f"for project '{slug}'. Wait for it to complete. "
            f"Then use SlackNotifyTool to send: "
            f"'✓ Discovery complete for {slug}. Starting Value Design.'"
        ),
        agent=agent,
        expected_output="Confirmation that discovery crew completed and Slack notified.",
    )

# create_run_value_design_task, create_run_architecture_task,
# create_run_delivery_task, create_run_business_plan_task — same pattern.
# Task 5 SlackNotifyTool message: "✓ {slug} pipeline complete. All outputs ready."
```

Each task takes `context_tasks=[previous_task]` so CrewAI passes prior results forward.

**Registry update (`agents/tools/registry.py`):**

The existing `"pam"` entry is updated from `[SQLiteStateTool, HumanInputTool]` to:


`HumanInputTool` removed — PAM does not prompt for human input directly.

## 7. PAM Crew and Orchestration Service

### `agents/crews/pam_crew.py`

```python
def create_pam_crew(
    slug: str,
    orchestration_run_id: int,
    llm_mode: str,
    llm: LLM | None = None,
) -> Crew:
    if llm is None:
        llm = get_pam_llm()  # Claude Opus 4.6

    tools = get_tools_for_agent("pam", slug=slug, run_id=orchestration_run_id)

    pam = create_pam_agent(slug=slug, llm=llm, tools=tools)

    t1 = create_run_discovery_task(agent=pam, slug=slug)
    t2 = create_run_value_design_task(agent=pam, slug=slug, context_tasks=[t1])
    t3 = create_run_architecture_task(agent=pam, slug=slug, context_tasks=[t2])
    t4 = create_run_delivery_task(agent=pam, slug=slug, context_tasks=[t3])
    t5 = create_run_business_plan_task(agent=pam, slug=slug, context_tasks=[t4])

    return Crew(
        agents=[pam],
        tasks=[t1, t2, t3, t4, t5],
        process=Process.sequential,
        verbose=True,
    )
```

### `run_service.py` — extract shared helper

Add `async def build_and_run_crew(slug: str, crew_name: str, run_id: int) -> Any`:
- Loads `config.yaml` for the project
- Dispatches the appropriate `create_*_crew()` factory (the per-crew `_run_*` private functions collapse into this single helper)
- Calls `await crew.kickoff_async()`
- Returns the result

`dispatch_crew()` is updated to call `build_and_run_crew()` internally. External interface unchanged.

### `api/services/orchestration_service.py`

```python
async def start_orchestration(slug: str) -> int:
    async with get_connection(slug) as conn:
        project = await fetch_project(conn, slug=slug)
        if not project:
            raise ValueError(f"Project '{slug}' not found")
        orchestration_run_id = await insert_orchestration_run(
            conn, project_id=project["id"]
        )
    asyncio.create_task(run_pam_crew(slug, orchestration_run_id))
    return orchestration_run_id

async def run_pam_crew(slug: str, orchestration_run_id: int) -> None:
    try:
        config = load_project_config(Path(settings.projects_dir) / slug)
        crew = create_pam_crew(
            slug=slug,
            orchestration_run_id=orchestration_run_id,
            llm_mode=config.get("llm_mode", "standard"),
        )
        await crew.kickoff_async()
        async with get_connection(slug) as conn:
            await update_orchestration_run_status(
                conn, run_id=orchestration_run_id, status="completed"
            )
    except Exception as e:
        async with get_connection(slug) as conn:
            await update_orchestration_run_status(
                conn, run_id=orchestration_run_id, status="failed"
            )
```

## 8. Database

**New table added to `api/database.py`:**

```sql
CREATE TABLE IF NOT EXISTS orchestration_runs (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id   INTEGER NOT NULL REFERENCES projects(id),
    status       TEXT NOT NULL DEFAULT 'running',
    started_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME
);
```

**Three new helpers:**
- `insert_orchestration_run(conn, *, project_id) -> int` — returns new id
- `update_orchestration_run_status(conn, *, run_id, status) -> None` — sets status + completed_at if terminal
- `fetch_orchestration_run(conn, *, run_id) -> dict | None`

Migration: added to the `init_db()` function alongside existing `CREATE TABLE IF NOT EXISTS` statements — safe to run on existing databases.

## 9. n8n Workflows

### `workflows/agentpool-notifications.json`

Handles both notification event types via a Switch node:

```
Webhook Trigger (POST /webhook/agentpool)
  └─ Switch on {{ $json.event_type }}
       ├─ "hitl_review"
       │    └─ Slack: Send DM to consultant
       │         Content: "Review needed for {{ $json.project_slug }}:\n
       │                   {{ $json.prompt }}\n
       │                   → {{ $json.review_url }}"
       └─ "crew_notification"
            └─ Slack: Post to channel {{ $json.slack_channel }}
                 Content: "{{ $json.message }}"
```

`N8N_WEBHOOK_URL` in `.env` points to this workflow's webhook URL. Covers both `HumanInputTool` (existing) and `SlackNotifyTool` (new) — no extra env var needed.

### `workflows/slack-run-command.json`

```
Slack Trigger (slash command: /run)
  └─ HTTP Request: POST http://host.docker.internal:8000/projects/{{ $json.text }}/orchestrate
  └─ Slack: Respond "⚙ Starting pipeline for {{ $json.text }}…"
```

`host.docker.internal` resolves to the Mac host from inside the n8n Docker container.

## 10. Slack App Setup

Manual steps (one-time per deployment):

1. Go to `api.slack.com/apps` → **Create New App → From scratch**
2. Name: `AgentPool`, workspace: your team workspace
3. **Slash Commands** → Create command:
   - Command: `/run`
   - Request URL: n8n webhook URL for `slack-run-command.json`
   - Short description: `Trigger full pipeline for a project`
4. **OAuth & Permissions** → Bot Token Scopes: `chat:write`, `commands`, `im:write`
5. **Install to Workspace** → copy **Bot User OAuth Token**
6. In n8n UI: **Credentials → New → Slack** → paste Bot Token
7. Update both workflow nodes to use the new Slack credential
8. Set in `.env`:
   ```
   N8N_WEBHOOK_URL=http://localhost:5678/webhook/agentpool
   ```
9. Activate both workflows in n8n
